# 阶段1：上下文收集

本阶段在任何审核逻辑执行之前运行，负责收集完整的审核上下文，供后续阶段（2-7）使用。

---

## 1. 审核模式识别

根据用户输入自动判断审核模式：

| 用户输入特征 | 审核模式 |
|---|---|
| 提供 git diff、PR/MR 链接、分支对比、提及 "PR"/"MR" | **PR增量审核** |
| 指向目录、文件列表、提及 "全量审核"/"代码审计" | **全量代码审计** |
| 无法判断 | 向用户确认，不要猜测 |

### PR增量审核

1. 通过 `git diff` 获取变更文件列表和 diff 内容
2. **先过滤排除文件**：从变更列表中移除文件名或文件夹名含 `test` 的文件、`wire_gen.go` 文件
3. 审核范围仅限于变更行及其上下文（前后各 ~10 行）
4. 特殊处理：
   - 新增文件 → 全文件审核
   - 修改文件 → 聚焦 diff 区域
   - 删除文件 → 仅检查是否有残留引用

### 全量代码审计

1. 扫描指定范围内所有 `.go` 文件
2. 排除项：`vendor/`、生成代码（`*.pb.go`、`*_gen.go`）、文件名或文件夹名包含 `test` 的文件（如 `*_test.go`、`testutil/`、`testdata/`、`internal/testing/` 等）、Wire 生成的文件（`wire_gen.go`）
3. 构建文件列表，按 package 结构组织

---

## 2. 调用 go-code-analyzer

使用 `go-code-analyzer` skill 对审核范围内的代码进行架构分析，获取以下三项输出：

1. **函数调用树** — 从入口点出发的完整调用层级
2. **服务依赖图** — 所有外部服务依赖（HTTP / gRPC / Kafka / Redis / MySQL 等）
3. **入口点列表** — HTTP 路由、MQ 消费者、Cron 任务、gRPC 方法

将分析结果作为架构上下文保存，供阶段 2-7 引用。

---

## 3. 规则加载

按以下顺序加载审核规则：

1. 检查项目根目录是否存在 `.go-review-rules.yaml`
2. **存在** → 解析文件内容，将自定义规则与文件中指定的基础规则集合并（自定义规则优先级更高）
3. **不存在** → 使用 `general` 作为默认规则集
4. 从 `references/builtin_rules.md` 加载对应的内置规则集

---

## 4. 严格度映射

对审核范围内的每个文件确定严格度等级，按优先级从高到低：

### 第一优先级：规则文件显式覆盖

检查 `.go-review-rules.yaml` 中的 `severity_overrides` 字段，使用路径 glob 匹配。

### 第二优先级：路径推断

若无显式覆盖，按路径模式推断：

| 路径模式 | 严格度 |
|---|---|
| `internal/core/`、`internal/service/`、`internal/domain/` | **strict** |
| `scripts/`、`tools/` | **relaxed** |
| 文件名或文件夹名含 `test` 的文件、`wire_gen.go` | **跳过（不审核）** |
| 其他所有文件 | **normal** |

### 第三优先级：维度级别调整

规则集中各审核维度的 per-dimension severity 会在文件级别严格度基础上进一步微调。

---

## 5. 代码类型识别（严格度自适应）

对代码进行分类以调整审核强度：

| 代码类型 | 典型路径/文件 | 审核强度 |
|---|---|---|
| 核心业务逻辑 | handler, service, domain | 最严格 |
| 数据访问层 | dao, repository, store | 严格 |
| 工具/辅助代码 | util, helper, middleware | 正常 |
| 配置/启动代码 | cmd, config | 宽松 |
| 测试代码 | `*_test.go`、`testutil/`、`testdata/`、文件夹名含 `test` | **跳过** |
| Wire 生成代码 | `wire_gen.go` | **跳过** |
| 生成代码 | `*.pb.go`, `*_gen.go` | **跳过** |

---

## 6. 输出格式

阶段1 产出一份上下文摘要，传递给所有后续阶段：

```
审核模式: {PR增量审核 | 全量代码审计}
审核范围: {文件列表}
规则集: {general | microservice | cli | 自定义}
架构上下文:
  入口点: [...]
  调用链摘要: {...}
  服务依赖: [...]
严格度映射:
  {file_path}: {strict | normal | relaxed}
  ...
```
