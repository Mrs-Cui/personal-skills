## 章节说明

### 时间戳格式
固定格式，机器可解析：
```
> 最后更新：2026-03-20 | 状态：✅ 有效
```

### 主要数据表 — 关键字段规则
- 列出有业务语义的字段：状态字段、金额字段、时间字段、外键 ID
- 忽略：`id`、`created_at`、`updated_at`、`deleted_at`

### 外部依赖 — 路径规则
- 内部服务：写相对于项目根的路径，如 `internal/services/award/`
- 外部 gRPC/HTTP：写包路径，如 `internal/secondpart/auth/`
- 引用说明：如另有 CLAUDE.md，注明 `← 见其 CLAUDE.md`

### Code Review 关键点 — 填写原则
- **不变式**：操作 A 必须同时触发操作 B（如"等级变更必须写 change log"）
- **高风险变更**：改了某个函数/文件，必须额外检查哪里
- **常见错误模式**：用 ❌ 开头，描述具体的错误写法（不是泛泛的"注意安全"）
