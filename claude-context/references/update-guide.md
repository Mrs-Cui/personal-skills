# Update 模式 — 章节更新策略

## 变更原因 → 更新章节对应表

| 变更原因关键词 | 更新章节 | 需要读取的文件 |
|-------------|---------|--------------|
| 新增表、新增字段、修改字段 | 主要数据表 | `*_mdl.go` |
| 新增接口、修改路由、删除接口 | API 端点 | `front.go` / `router.go` |
| 业务规则变更、流程变化 | 核心业务流程 | `service/core.go` 等 |
| 新增外部服务调用 | 外部依赖 | `provider.go` 及调用处 |
| 发现新的 bug 模式 | Code Review 关键点 — 常见错误模式 | 无需读文件，用户描述即可 |
| 发现新的约束 | Code Review 关键点 — 不变式 | 相关业务逻辑文件 |

---

## 保护规则（不可覆盖的内容）

以下内容是开发者手动积累的，**永远不覆盖**：

1. `### 常见错误模式` 下已有的 ❌ 条目
2. `### 不变式` 下已有的非"暂无"条目
3. `## 业务说明` 中已有的人工描述（比 AI 生成的更准确）

**规则**：如果现有内容不是"暂无"/"待补充"/"待开发过程中补充"，则保留现有内容，不替换。

---

## 时间戳更新规则

无论更新了哪些章节，都必须同步更新文件头时间戳：

```
> 最后更新：<今天日期> | 状态：✅ 有效
```

---

## 冲突处理

当代码与现有 CLAUDE.md 描述不一致时：

- **表结构冲突**：以代码（`*_mdl.go`）为准，更新文档
- **API 端点冲突**：以代码（路由注册文件）为准，更新文档
- **业务规则冲突**：**不自动覆盖**，输出提示：
  ```
  ⚠️  发现潜在冲突：
  文档描述：xxx
  代码实现：yyy
  请人工确认哪个是正确的，然后手动更新
  ```
